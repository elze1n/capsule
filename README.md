# Capsule 

![capsule banner](assets/imgs/capsule_banner.png)

## Overview
**Capsule** is a modern Android application designed to simplify doctor–patient interactions by providing an integrated platform for booking appointments, online consultations via chat, and secure electronic prescriptions.  
The application enhances healthcare accessibility by enabling users to manage consultations and prescriptions in one convenient place.

## Objectives
- Enable patients to search for doctors, view availability, and book appointments easily.
- Provide secure chat-based online consultations between doctors and patients.
- Generate electronic prescriptions in a clear and structured format for pharmacy use.
- Offer doctors an interface to manage schedules, consultations, and prescriptions.
- Ensure data security and trust through authentication and encryption.

---

## Stakeholder Analysis

| Stakeholder | Interest | Needs |
|------------|--------|-------|
| Patients | Easy access to doctors and prescriptions | Book appointments, chat with doctors, receive prescriptions |
| Doctors | Efficient patient management | Manage schedules, consultations, and issue e-prescriptions |

---

## User Requirements
- Patients can create an account and log in securely.
- Patients can search for doctors by specialization or name.
- Patients can view doctors’ availability and book appointments.
- Patients can chat with doctors for online consultation.
- Patients can view electronic prescriptions.
- Doctors can log in and manage their profiles.
- Doctors can view, mark as completed, or cancel appointments.
- Doctors can create and send electronic prescriptions.
- Users can update their personal information.

---

## System Requirements
- Android mobile application.
- Firebase Authentication for user login.
- Firebase Firestore / Realtime Database.
- Cloud Storage for profile images and prescriptions.
- Internet connectivity.
- Secure backend services.

---

## Functional Requirements
- The system shall allow users to register and authenticate securely.
- The system shall allow patients to search for doctors by specialization.
- The system shall allow patients to book and cancel appointments.
- The system shall allow doctors to manage appointment schedules.
- The system shall support real-time chat between doctors and patients.
- The system shall allow doctors to create electronic prescriptions.
- The system shall store prescriptions securely and allow patient access.
- The system shall send notifications for appointment updates.

---

## Non-Functional Requirements

### Performance
- The system shall respond to user actions within 2 seconds.

### Security
- User data shall be encrypted and protected using secure authentication.

### Usability
- The application shall provide a user-friendly and intuitive interface.

### Reliability
- The system shall be available 24/7 with minimal downtime.

### Scalability
- The system shall support an increasing number of users without performance degradation.

### Maintainability
- The system shall be easy to maintain and update.

---

## Conclusion
Capsule aims to improve healthcare accessibility by providing an efficient and secure platform for doctor–patient communication, appointment management, and electronic prescriptions.
